<task type="auto">
  <name>Task 2: Implement full index command pipeline</name>
  <files>src/cli/commands/index-cmd.ts</files>
  <action>
    Replace the stub in `src/cli/commands/index-cmd.ts` with the full implementation. All imports must use dynamic `await import()` for lazy loading (matching the established pattern).

    **Function signature:**
    ```typescript
    export async function runIndex(
      targetPath: string,
      options: { ignore: boolean; type?: string; quiet?: boolean; clear?: boolean; pretty?: boolean }
    ): Promise<void>
    ```

    **Early exit for unsupported types:**
    If `options.type` is `'text'` or `'image'`, output a structured JSON error to stdout and return:
    `{ "status": "error", "message": "text pipeline not yet implemented (Phase 5)" }`
    (same for image). Do NOT throw -- just print and exit cleanly.

    **Pipeline flow (in order):**

    1. **Resolve path:** `const absPath = path.resolve(targetPath)`

    2. **Open vector collections:** Dynamic import `vector-db.ts`, call `openProjectCollections(absPath)` to get `{ col768, storagePath }`.

    3. **Handle --clear:** If `options.clear`:
       - Dynamic import `manifest-cache.ts`, call `clearManifest(absPath)` to delete `.ez-search-cache`
       - Call `col768.remove()` is wrong -- instead, need to destroy and recreate. The research says `--clear` must delete both Zvec collections AND manifest cache. For Zvec, the simplest approach: use `rmSync` to delete the `col-768` subdir inside `storagePath`, then re-call `openProjectCollections(absPath)` to recreate fresh. Import `rmSync` from 'fs'.
       - Actually, re-read vector-db.ts -- `destroySync()` deletes from disk but is on the handle. The cleanest approach: `rmSync(path.join(storagePath, 'col-768'), { recursive: true, force: true })` then re-open. But `openProjectCollections` creates both cols. So: if `--clear`, do `rmSync` on the col-768 dir, `rmSync` on col-512 dir, `rmSync` the schema-version.json, then call `openProjectCollections` again to get fresh collections.
       - Update: Wipe the ENTIRE storagePath (including schema-version.json) with `rmSync(storagePath, { recursive: true, force: true })`, then re-call `openProjectCollections(absPath)` which will `mkdirSync` and create fresh collections.

    4. **Load manifest:** Dynamic import `manifest-cache.ts`, call `loadManifest(absPath)`.

    5. **Scan files:** Dynamic import `file-scanner.ts`, call `scanFiles(absPath, { useIgnoreFiles: options.ignore, typeFilter: options.type as FileType | undefined })`. Collect all files into an array (the scanner is an async generator).
       - If `options.type` is undefined, default to scanning code files only for Phase 3. Use `typeFilter: (options.type as FileType | undefined) ?? 'code'`.

    6. **Detect changes:** Compare scanned files against manifest:
       - For each file, check if manifest has an entry with matching `mtime` + `size` (fast path -- skip).
       - If mtime or size changed, read file content and hash with `hashContent()`. If hash matches manifest, skip (mtime-only change like touch/checkout).
       - Files in scan but not in manifest: new files (must index).
       - Files in manifest but not in scan: deleted files (must remove chunks from Zvec).

    7. **Handle deletions:** For each deleted file path, iterate its `manifest.files[path].chunks` and call `col768.remove(chunk.id)` for each chunk. Then delete the entry from `manifest.files`.

    8. **Process changed/new files:**
       - Load tokenizer once: dynamic import `chunker.ts`, call `loadTokenizer()`.
       - Load embedding pipeline once: dynamic import `model-router.ts`, call `createEmbeddingPipeline('code')`.
       - For each changed file:
         a. Read file content with `fsp.readFile(file.absolutePath, 'utf8')`
         b. Chunk with `chunkFile(content, tokenizer)`
         c. Compute chunk metadata: for each chunk, `makeChunkId(file.relativePath, chunk.chunkIndex)`, `hashText(chunk.text)`
         d. **Chunk-level diff (if file existed before):** Compare new chunk textHashes against old `manifest.files[file.relativePath].chunks` by index. Only re-embed chunks whose textHash changed or are new. Remove old chunks that no longer exist (file got shorter).
         e. Collect chunks needing embedding into a batch buffer.

    9. **Batch embedding:** Collect all chunks needing embedding across all files. Process in batches of 32:
       - `const BATCH_SIZE = 32`
       - For each batch, extract texts, call `pipe.embed(texts)` to get `Float32Array[]`
       - For each embedding, call `col768.insert(chunkId, embedding, { filePath: relPath, chunkIndex, modelId: pipe.modelId, lineStart, lineEnd, chunkText: chunk.text })`

    10. **Update manifest:** For each processed file, update `manifest.files[relPath]` with new `ManifestEntry`: `{ mtime: file.mtimeMs, size: file.sizeBytes, hash: fileHash, chunks: [...chunkRecords] }`.

    11. **Optimize:** Call `col768.optimize()` once after all inserts/deletes.

    12. **Save manifest:** Call `saveManifest(absPath, manifest)`. This MUST happen AFTER optimize succeeds.

    13. **Dispose pipeline:** Call `pipe.dispose()`.

    14. **Output results:**
        Track stats throughout: `filesScanned`, `filesIndexed`, `filesSkipped`, `chunksCreated`, `chunksReused`, `chunksRemoved`, `durationMs`.

        Build output object:
        ```
        {
          status: filesIndexed === 0 && deletedCount === 0 ? 'no_changes' : 'ok',
          path: absPath,
          filesScanned, filesIndexed, filesSkipped,
          chunksCreated, chunksReused, chunksRemoved,
          durationMs: Date.now() - startTime,
          storageDir: storagePath
        }
        ```

        If `options.quiet`: no output.
        If `options.pretty`: print human-readable summary to stdout (e.g., "Indexed 38 files, 156 new chunks in 4.2s").
        Otherwise: `console.log(JSON.stringify(output))`.

    **Error handling:** Wrap the entire pipeline in try/catch. On error, output `{ "status": "error", "message": error.message }` to stdout (JSON) or stderr (pretty). Do NOT write manifest on error.

    **Import notes:**
    - Use `import * as path from 'path'` and `import * as fsp from 'fs/promises'` at top level (these are lightweight).
    - All service imports MUST use `await import()` inside the function body for lazy loading.
    - Import `rmSync` from 'fs' at top level (lightweight, needed for --clear).
