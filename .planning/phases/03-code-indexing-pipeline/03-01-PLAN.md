<task type="auto">
  <name>Task 2: Create manifest cache service</name>
  <files>src/services/manifest-cache.ts</files>
  <action>
    Create `src/services/manifest-cache.ts` implementing the manifest cache for incremental indexing. Follow the research patterns exactly:

    **Types to export:**
    - `ChunkRecord`: `{ id: string, lineStart: number, lineEnd: number, tokenCount: number, textHash: string }`
    - `ManifestEntry`: `{ mtime: number, size: number, hash: string, chunks: ChunkRecord[] }`
    - `Manifest`: `{ version: number, files: Record<string, ManifestEntry> }`

    **Functions to export:**

    1. `loadManifest(projectDir: string): Manifest` -- Read `.ez-search-cache` from projectDir. Return empty manifest `{ version: 1, files: {} }` if file doesn't exist, is corrupt JSON, or has wrong version. Use `readFileSync` + `existsSync`.

    2. `saveManifest(projectDir: string, manifest: Manifest): void` -- Write manifest atomically: `writeFileSync` to `.ez-search-cache.tmp`, then `renameSync` to `.ez-search-cache`. This prevents partial writes on crash.

    3. `clearManifest(projectDir: string): void` -- Delete `.ez-search-cache` if it exists. Use `unlinkSync` wrapped in try/catch.

    4. `hashContent(content: Buffer): string` -- `crypto.createHash('sha256').update(content).digest('hex').slice(0, 16)`.

    5. `hashText(text: string): string` -- Same as hashContent but accepts string.

    6. `makeChunkId(relativeFilePath: string, chunkIndex: number): string` -- `crypto.createHash('sha256').update(relativeFilePath).digest('hex').slice(0, 12) + '_' + String(chunkIndex).padStart(4, '0')`. No colons (Zvec constraint).

    **Constants:**
    - `MANIFEST_VERSION = 1`
    - `MANIFEST_FILENAME = '.ez-search-cache'`

    Use `import crypto from 'node:crypto'` and fs imports. Do NOT use `require`. Follow ESM conventions matching the rest of the codebase.
  </action>
  <verify>
    Run `npx tsc --noEmit` -- no type errors. Verify the file exports all 6 functions and 3 types by grepping for `export`.
  </verify>

  </done>
