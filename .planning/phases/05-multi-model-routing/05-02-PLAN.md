<objective>
Create the CLIP image embedding service.

Purpose: Image files (.jpg, .png, .webp) need a completely different pipeline from text/code. Instead of tokenizing text, we load the image, preprocess it with CLIP's AutoProcessor, and run it through CLIPVisionModelWithProjection to get a 512-dim embedding. One image = one vector (no chunking). The CLIP quantized models fail in Node.js onnxruntime-node (ConvInteger operator not implemented), so we MUST use `dtype: 'fp32'`.

Output: `src/services/image-embedder.ts` with `createImageEmbeddingPipeline()` that returns an `ImageEmbeddingPipeline` with an `embedImage(absolutePath)` method.
</objective>

**`createImageEmbeddingPipeline(): Promise<ImageEmbeddingPipeline>`:**

1. Set `env.cacheDir = resolveModelCachePath()` and `env.allowRemoteModels = true`
2. Load processor: `AutoProcessor.from_pretrained(CLIP_MODEL_ID)`
3. Load vision model: `CLIPVisionModelWithProjection.from_pretrained(CLIP_MODEL_ID, { dtype: 'fp32' })`
   - CRITICAL: Must use `dtype: 'fp32'` â€” quantized (int8, uint8) fails with "ConvInteger(10) is not implemented" in onnxruntime-node.
4. Load both in parallel with `Promise.all`
5. Log to stderr: `[image-embedder] Loaded CLIP vision model (fp32)`

Return object implementing ImageEmbeddingPipeline:

**`embedImage(absolutePath: string)`:**
1. Load image: `const image = await RawImage.fromURL(absolutePath)` (works with file:// URLs and absolute paths â€” use `'file://' + absolutePath` if bare path fails; test both)
2. Process: `const inputs = await processor(image)`
3. Run model: `const output = await visionModel(inputs)`
4. Extract embedding from `output.image_embeds` â€” this is a Tensor. Access `.data` for the Float32Array, then slice to first 512 values (one image = one vector).
5. Return as `new Float32Array(output.image_embeds.data.slice(0, CLIP_DIM))`

**`dispose()`:**
- Call dispose on visionModel if available

Follow existing code style from model-router.ts: JSDoc comments, section separators, stderr logging for model loading.

**IMPORTANT:** The RawImage.fromURL may need a file:// prefix for local paths. Try bare path first; if it fails, prepend `file://`. Document whichever works in a comment.
