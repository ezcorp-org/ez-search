<objective>
Create the text chunking service for Nomic text pipeline.

Purpose: Text documents (.md, .txt, .pdf, .csv, .rst) need different chunking than code. Code uses token-window sliding (chunker.ts with Jina tokenizer). Text should use paragraph-boundary chunking: split on `\n\n`, merge small paragraphs up to ~1600 chars (~400 tokens for Nomic), split oversized paragraphs. PDF extraction via pdf-parse converts binary PDF to plain text before chunking.

Output: `src/services/text-chunker.ts` with `chunkTextFile()` and `extractPdfText()` exports. `pdf-parse` added to package.json dependencies.
</objective>

  <action>
1. Install pdf-parse: `bun add pdf-parse` (pure JS PDF text extraction, no native deps).

2. Create `src/services/text-chunker.ts` with the following:

**Constants:**
- `MAX_CHUNK_CHARS = 1600` (approx 400 Nomic tokens)
- `MIN_CHUNK_CHARS = 200` (don't create tiny fragments)

**`extractPdfText(buffer: Buffer): Promise<string>`**
- Dynamically import pdf-parse: `const pdf = (await import('pdf-parse')).default`
- Call `pdf(buffer)` and return `result.text`
- Wrap in try/catch, throw descriptive error on failure

**`chunkTextFile(text: string): TextChunk[]`**
- Return type `TextChunk` with fields: `{ text: string, chunkIndex: number, charCount: number }`
- No line tracking needed for text docs (unlike code) — text results show file path only.

Algorithm:
1. Split text on paragraph boundaries: `text.split(/\n\n+/)`
2. Filter out empty strings after split
3. Merge pass: iterate paragraphs, accumulate into current chunk. When adding next paragraph would exceed MAX_CHUNK_CHARS, finalize current chunk and start new one.
4. Split pass: if any single paragraph exceeds MAX_CHUNK_CHARS, split it on sentence boundaries (`. ` or `.\n`). If a sentence still exceeds MAX_CHUNK_CHARS, hard-split at MAX_CHUNK_CHARS.
5. Final filter: drop any chunks under MIN_CHUNK_CHARS (unless it's the only chunk)
6. Assign sequential chunkIndex starting at 0

Export the `TextChunk` interface, `chunkTextFile`, and `extractPdfText`.

Follow existing code style from chunker.ts: JSDoc comments on exports, section separators with `// -- Name ---`.
