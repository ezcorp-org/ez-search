  <name>Task 1: Implement Zvec collection wrapper service</name>
  <files>
    src/services/vector-db.ts
  </files>
  <action>
    Create `src/services/vector-db.ts` that wraps Zvec operations behind a clean interface.

    Implementation:

    1. Use `createRequire(import.meta.url)` to import @zvec/zvec (CJS package in ESM project)
    2. Call `ZVecInitialize({ logLevel: ZVecLogLevel.WARN })` at module level

    3. Define `VectorCollection` interface/type with methods:
       - `insert(id: string, embedding: Float32Array, metadata: Record<string, string | number>): void`
       - `query(embedding: Float32Array, topK: number): Array<{ id: string; distance: number; metadata: Record<string, string | number> }>`
       - `remove(id: string): void`
       - `optimize(): void`  (calls optimizeSync — required after bulk inserts for 10x query speedup)
       - `close(): void`

    4. Create a helper `createCollection(storageDir: string, name: string, dim: number): VectorCollection` that:
       - Creates schema: vector VECTOR_FP32, {dim}-dim, HNSW index, COSINE metric, m=50, efConstruction=500
       - Fields: filePath (STRING), chunkIndex (INT32), modelId (STRING), lineStart (INT32), lineEnd (INT32)
       - Opens with `ZVecCreateAndOpen(path.join(storageDir, name), schema)`
       - Wraps the raw Zvec handle in a `VectorCollection` object

    5. `openProjectCollections(projectDir: string)` function:
       - Import `resolveProjectStoragePath` from `../config/paths.js`
       - Compute storage path from projectDir
       - Create parent directory with `mkdirSync(storageDir, { recursive: true })`
       - Create 768-dim collection: `createCollection(storageDir, 'col-768', 768)`
       - Create 512-dim collection: `createCollection(storageDir, 'col-512', 512)`
       - Return `{ col768: VectorCollection, col512: VectorCollection, storagePath: storageDir }`

    6. For the `query` method, remember: COSINE metric returns distance (0 = exact match, ascending = less similar). Do NOT invert scores — consumers handle ranking.

    7. For `insert`, validate that id does NOT contain colons (throw descriptive error if it does). IDs must use underscores/hyphens only.

    Critical notes from spike:
    - ZVecCreateAndOpen fails if parent dir doesn't exist — always mkdirSync first
    - optimizeSync must be called after bulk inserts before queries are fast
    - COSINE returns distance, not similarity
    - IDs with colons are rejected by Zvec
  </action>

  <name>Task 2: Implement model router with WebGPU-to-CPU fallback</name>
  <files>
    src/services/model-router.ts
  </files>
  <action>
    Create `src/services/model-router.ts` that wraps Transformers.js pipeline creation with WebGPU fallback.

    Implementation:

    1. Import `pipeline`, `env` from '@huggingface/transformers'
    2. Import `resolveModelCachePath` from '../config/paths.js'
    3. Import `ModelBackend` from '../types.js'

    4. Define model registry constant:
       ```typescript
       const MODEL_REGISTRY = {
         code: { id: 'jinaai/jina-embeddings-v2-base-code', dim: 768 },
         text: { id: 'nomic-ai/nomic-embed-text-v1.5', dim: 768, taskPrefix: { document: 'search_document: ', query: 'search_query: ' } },
       } as const;
       ```

    5. Define `EmbeddingPipeline` type/interface:
       - `embed(texts: string[]): Promise<Float32Array[]>` — returns one Float32Array per input text
       - `backend: ModelBackend` — which backend is active
       - `modelId: string` — which model is loaded
       - `dim: number` — embedding dimension
       - `dispose(): Promise<void>` — cleanup

    6. `createEmbeddingPipeline(modelType: 'code' | 'text', options?: { progressCallback?: Function })`:
       - Set `env.cacheDir = resolveModelCachePath()` BEFORE first pipeline() call
       - Set `env.allowRemoteModels = true`
       - Get model info from MODEL_REGISTRY
       - Try WebGPU first: `pipeline('feature-extraction', modelId, { device: 'webgpu', dtype: 'fp32', progress_callback })`
       - On catch, fall back to CPU: `pipeline('feature-extraction', modelId, { device: 'cpu', dtype: 'q8', progress_callback })`
       - Log backend to stderr: `console.error('Using WebGPU')` or `console.error('Using CPU')`
       - Return `EmbeddingPipeline` wrapper around the raw pipeline

    7. The `embed` method:
       - Takes array of strings
       - Calls the pipeline with `{ pooling: 'mean', normalize: true }`
       - Extracts Float32Array from each output tensor via `.tolist()` or `.data`
       - Returns `Float32Array[]` — one per input

    Critical notes from spike:
    - device: 'cpu' NOT 'wasm' (wasm is browser-only)
    - Official model IDs only (jinaai/, nomic-ai/) — Xenova/ returns 401
    - env.cacheDir must be set BEFORE first pipeline() call
    - Nomic text model requires task prefixes (search_document: / search_query:) — but the pipeline itself doesn't add them. The caller is responsible. Document this in a JSDoc comment.
    - Progress callback fields may be missing — guard all accesses
  </action>
