**1. Create `src/cli/errors.ts`** — shared structured error utility:

- Export `ErrorCode` type: `'NO_INDEX' | 'EMPTY_DIR' | 'UNSUPPORTED_TYPE' | 'CORRUPT_MANIFEST' | 'GENERAL_ERROR'`
- Export `StructuredError` interface: `{ error: true; code: ErrorCode; message: string; suggestion: string }`
- Export `emitError(opts: { code: ErrorCode; message: string; suggestion: string }, format: 'json' | 'text', exitCode?: number): never`
  - `format === 'text'`: write `Error: {message}. Try: {suggestion}` to stderr
  - `format === 'json'`: write `JSON.stringify({ error: true, code, message, suggestion })` to stdout
  - Call `process.exit(exitCode)` (default 1)
  - Return type `never` so TypeScript knows control flow ends

**2. Replace stub in `src/cli/commands/status-cmd.ts`**:

The function signature becomes: `runStatus(options: { format?: string; ignore: boolean }): Promise<void>`

Implementation steps (all using dynamic imports for lazy loading):

a. **Check manifest exists**: `existsSync(path.join(process.cwd(), MANIFEST_FILENAME))`. If not, call `emitError({ code: 'NO_INDEX', message: 'No index found in current directory', suggestion: 'Run: ez-search index .' }, format, 2)`.

b. **Load manifest**: Call `loadManifest(projectDir)`. Detect corruption: if manifest file exists AND `Object.keys(manifest.files).length === 0` AND file size > 10 bytes, set a `warning` string: `"Manifest appears corrupt or version-mismatched. Reported data may be incomplete."` and `suggestion: "Run: ez-search index --clear ."`.

c. **Get lastIndexed**: `statSync(manifestPath).mtimeMs` -> `new Date(mtimeMs).toISOString()`.

d. **Per-type counts**: Loop `Object.entries(manifest.files)`, look up `path.extname(relPath).toLowerCase()` in `EXTENSION_MAP` (imported from `../../types.js`). Build `byType: { code: { files, chunks }, text: { files, chunks }, image: { files, chunks } }`. All three types always present (even zeros) for predictable agent parsing. Derive `modelTypes` as only the non-zero types.

e. **Index size**: Create `calcDirSize(dir: string): Promise<number>` helper. Use `fsp.readdir(dir, { recursive: true, withFileTypes: true })`, sum `fsp.stat(path.join(entry.parentPath, entry.name)).size` for files only. Return 0 if dir doesn't exist or on error. Use `resolveProjectStoragePath(projectDir)` to get the storage dir. Check `existsSync(storagePath)` first — if missing but manifest exists, emit CORRUPT_MANIFEST error: `{ code: 'CORRUPT_MANIFEST', message: 'Manifest exists but vector storage is missing', suggestion: 'Run: ez-search index --clear .' }` with exit code 1.

f. **Staleness detection**: Create `calcStaleness(projectDir, manifest, useIgnoreFiles): Promise<number>`. Import `scanFiles` from `../../services/file-scanner.js`. Count files where: (1) file is in scan but not in manifest (new), or (2) file is in scan and manifest but mtime differs. ALSO count deleted files: iterate `manifest.files`, check `existsSync(path.join(projectDir, relPath))` for each — if missing, count as stale.

g. **Output JSON** (default): `JSON.stringify` the full status object: `{ fileCount, chunkCount, lastIndexed, modelTypes, indexSizeBytes, storagePath, staleFileCount, byType }`. If `warning` was set, include `warning` and `suggestion` fields.

h. **Output text** (`--format text`):
```
Index: {storagePath}
Files: {fileCount} (code: N, text: N, image: N)
Chunks: {chunkCount}
Last indexed: {lastIndexed}
Index size: {human-readable size, e.g. "8.3 MB"}
Stale files: {staleFileCount}
```
If warning, append: `Warning: {warning}`

For human-readable size: bytes < 1024 -> "N B", < 1MB -> "N.N KB", < 1GB -> "N.N MB", else "N.N GB".

```
    const { runStatus } = await import('./commands/status-cmd.js');
    await runStatus(options);
  });
```

**IMPORTANT anti-patterns to avoid:**
- Do NOT call `openProjectCollections()` — it creates directories as a side effect. Use `resolveProjectStoragePath` + `existsSync` only.
- Do NOT call `loadManifest` before checking file existence — it silently returns empty manifest for missing files.
- JSON errors go to stdout (same channel as normal output for agent parsing). Text errors go to stderr.
