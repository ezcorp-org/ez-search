<task type="auto">
  <name>Task 1: Initialize project and Zvec spike script</name>
  <files>package.json, tsconfig.json, spike/zvec-spike.ts</files>
  <action>
1. Initialize the project if package.json doesn't exist: `npm init -y` and set `"type": "module"` in package.json.

2. Create tsconfig.json with:
   - target: ES2022, module: NodeNext, moduleResolution: NodeNext
   - strict: true, outDir: dist, rootDir: .
   - esModuleInterop: true

3. Install dependencies: `npm install @zvec/zvec typescript tsx`
   - tsx is for running TypeScript directly during spikes

4. Create `spike/zvec-spike.ts` that validates ALL of the following in sequence:

   a. **Collection creation** with realistic schema:
      - name: 'spike-validation'
      - vector field: 'embedding', FP32, dimension 768, HNSW index, COSINE metric, m=50, efConstruction=500
      - string fields: 'path' and 'content'
      - Use ZVecCreateAndOpen with path `/tmp/ez-search-zvec-spike`

   b. **Bulk insert 1000 vectors** at dimension 768:
      - Generate 1000 vectors with random Float32Array(768) values
      - Each entry gets id like `doc:${i}`, path like `src/file-${i}.ts`, content like `content for file ${i}`
      - Time the total insert with `performance.now()` and log it
      - Verify all 1000 insertSync calls return `{ ok: true }`

   c. **Query by similarity**:
      - Query using the vector from doc:0 as the query vector, topk=10, outputFields=['path', 'content']
      - Verify doc:0 appears in results (should be score ~1.0 since it's an exact match)
      - Verify results are ranked by score descending
      - Log timing for the query

   d. **Test optimizeSync effect**:
      - Query BEFORE optimizeSync, record results and timing
      - Call optimizeSync, record timing
      - Query AFTER optimizeSync with same vector, record results and timing
      - Log whether results or timing changed

   e. **Delete and verify**:
      - Delete doc:0 by ID using deleteSync
      - Query again with same vector -- doc:0 must NOT appear in results
      - Delete by filter: deleteByFilterSync("path = 'src/file-1.ts'") -- doc:1 must also disappear

   f. **Cleanup**: Call destroySync to remove test data from /tmp

   g. **Summary output**: Print a clear PASS/FAIL verdict for each operation, plus all timing numbers.

   Use `isZVecError` for error checking. Wrap the whole thing in a try/catch that prints the full error if anything fails.

   IMPORTANT: Use the exact API from the research doc -- methods are `insertSync`, `querySync`, `deleteSync`, `deleteByFilterSync`, `optimizeSync`, `destroySync`. Vector values can be plain number arrays (they accept number[] | Float32Array).
  </action>
  <verify>
Run: `npx tsx spike/zvec-spike.ts`

Expected: All operations print PASS. 1000 vectors inserted, queries return ranked results, deletes remove entries, timing numbers are logged.

If any operation fails with a Zvec error, the error code and message are printed clearly.
  </verify>
