<task type="auto">
  <name>Task 1: Transformers.js inference spike with WebGPU fallback</name>
  <files>spike/transformers-spike.ts, package.json</files>
  <action>
1. Install transformers.js if not already present: `npm install @huggingface/transformers@next`

2. Create `spike/transformers-spike.ts` that validates ALL of the following:

   a. **WebGPU-to-CPU fallback pipeline factory**:
      - Implement a `createPipeline(modelId, options?)` function that:
        - Tries `pipeline('feature-extraction', modelId, { device: 'webgpu', dtype: 'fp32' })` first
        - On failure, catches the error, logs it, falls back to `pipeline('feature-extraction', modelId, { device: 'cpu', dtype: 'q8' })`
        - Returns `{ pipe, backend: 'webgpu' | 'cpu' }`
      - Log which backend was selected

   b. **Jina code model validation** (`jinaai/jina-embeddings-v2-base-code`):
      - Load model using the fallback factory
      - Time the model loading with `performance.now()`
      - Generate embeddings for these test inputs:
        ```
        "function add(a: number, b: number) { return a + b; }"
        "function sum(x: number, y: number) { return x + y; }"
        "class DatabaseConnection { constructor(private url: string) {} }"
        ```
      - Use `{ pooling: 'mean' }` for pipeline options
      - Time the inference
      - Verify output dimensions are [1, 768] for each input
      - Compute cosine similarity between add/sum (should be HIGH, > 0.8) and between add/DatabaseConnection (should be LOWER)
      - Log all similarity scores

   c. **Nomic text model validation** (`nomic-ai/nomic-embed-text-v1.5`):
      - Load model using the fallback factory
      - Generate embeddings with task prefixes:
        ```
        "search_document: How to authenticate users with JWT tokens"
        "search_query: jwt authentication"
        "search_document: Recipe for chocolate cake with frosting"
        ```
      - Use `{ pooling: 'mean', normalize: true }` for pipeline options
      - Verify output dimensions are [1, 768]
      - Compute cosine similarity: jwt_doc vs jwt_query (should be HIGH) vs cake_doc (should be LOW)
      - Log all similarity scores

   d. **Cosine similarity helper**:
      - Import `cos_sim` from `@huggingface/transformers` if available
      - Otherwise implement: `dot(a,b) / (norm(a) * norm(b))`
      - The embedding output from pipeline is a Tensor; access raw data via `.data` property (Float32Array) or `.tolist()` to get nested arrays

   e. **Summary output**: Print clear results:
      - Backend selected (webgpu or cpu)
      - Model load times
      - Inference times per input
      - Embedding dimensions confirmed
      - Similarity scores with PASS/FAIL for each pair
      - Overall PASS/FAIL verdict

   IMPORTANT notes from research:
   - Use `device: 'cpu'` NOT `device: 'wasm'` for Node.js fallback
   - Use official model IDs (jinaai/, nomic-ai/) NOT Xenova/ (returns 401)
   - nomic model REQUIRES task prefixes ("search_document:", "search_query:")
   - Pipeline output is a Tensor object; use `.data` for raw Float32Array or `.tolist()` for nested array
   - First model load downloads the model (~100-300MB); subsequent runs use cache
  </action>
