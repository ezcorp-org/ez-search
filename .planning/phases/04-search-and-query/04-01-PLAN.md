<task type="auto">
  <name>Task 1: Implement query pipeline in query-cmd.ts</name>
  <files>src/cli/commands/query-cmd.ts</files>
  <action>
Replace the stub in query-cmd.ts with the full query pipeline. The function signature changes to accept the updated options shape (format, threshold added; pretty removed).

Pipeline steps:
1. Resolve the project directory (use `process.cwd()` as the project root -- same as index command uses targetPath but query doesn't take a path arg, it searches the current project).
2. Open vector collections via `openProjectCollections(projectDir)` -- dynamic import like index-cmd does.
3. Load manifest via `loadManifest(projectDir)` to get `totalIndexed` (count of `Object.keys(manifest.files).length`).
4. Create embedding pipeline via `createEmbeddingPipeline('code')` -- dynamic import.
5. Embed the query text: `const [queryEmbedding] = await pipe.embed([text])`.
6. Determine fetch count: when `--dir` or `--threshold` are active, fetch `topK * 3` from Zvec to account for post-filtering. Otherwise fetch `topK`.
7. Call `col768.query(queryEmbedding, fetchCount)` to get raw results.
8. Normalize scores: `Math.round(Math.max(0, Math.min(1, 1 - result.distance)) * 10000) / 10000` (Zvec COSINE returns distance where 0=exact match).
9. Apply `--threshold` filter: exclude results where normalized score < threshold.
10. Apply `--dir` filter: normalize the dir option by stripping leading `./` and trailing `/`, then filter to results where `metadata.filePath.startsWith(normalizedDir)`.
11. Collapse adjacent chunks: group results by `metadata.filePath`, sort each group by `metadata.chunkIndex` (number), merge consecutive chunkIndex values into single results with: merged lineStart (min), merged lineEnd (max), max score of group, concatenated chunkText (joined with newline). Then flatten back to a single array sorted by score descending.
12. Slice to `topK` limit.
13. Dispose the embedding pipeline.
14. Build output object:
    ```
    {
      query: text,
      results: results.map(r => ({
        file: r.filePath,
        lines: { start: r.lineStart, end: r.lineEnd },
        score: r.score,
        text: r.chunkText
      })),
      totalIndexed: manifestFileCount,
      searchScope: options.dir || '.'
    }
    ```
15. If `--format text`: output each result as `File: <file> | Lines: <start>-<end> | Relevance: <score>` followed by indented chunk text (each line prefixed with 4 spaces). If results are empty, print "No results found."
16. Otherwise (default JSON): `console.log(JSON.stringify(output, null, 2))`.
17. If no results, JSON output should include an empty results array and a `message: "No results found"` field.

Error handling:
- Wrap the entire pipeline in try/catch. On error, output `{ query: text, error: message }` as JSON to stdout (or plain text to stderr if --format text).
- If Zvec query throws (e.g., empty collection), catch it and return empty results with message.

Important implementation notes:
- Use camelCase for JSON fields (matches existing codebase convention in index-cmd.ts).
- Follow the dynamic import pattern from index-cmd.ts: `const { openProjectCollections } = await import('../../services/vector-db.js')`.
- The options type should be: `{ format?: string; topK: string; dir?: string; threshold?: string }`.
- Parse topK and threshold from strings to numbers: `const topK = parseInt(options.topK, 10)` and `const threshold = options.threshold ? parseFloat(options.threshold) : undefined`.
- For the adjacent chunk collapsing, "consecutive" means chunkIndex values that differ by exactly 1. If a file has chunks at indices 0,1,2 and 5, collapse 0-1-2 into one group and keep 5 separate.
